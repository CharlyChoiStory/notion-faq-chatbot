# 성형외과 FAQ 챗봇 하네스 구축 데이터 명세서

## 0. 문서 개요

이 문서는 **소규모 성형외과 고객응대 AI 챗봇**을 구축할 때, 기존 FAQ 문서와 온토로지 규칙을 기반으로 **라우팅/하네스 레이어**를 추가하기 위한 데이터 명세서입니다.

목표는 단순 FAQ 검색 챗봇이 아니라 다음 기능을 갖춘 실전형 시스템을 만드는 것입니다.

- 고객 질문의 시술명, 증상, 문의 유형 분류
- 벡터 DB 기반 관련 FAQ 검색
- 온토로지 기반 위험도 판단
- 자동 답변 가능 여부 판단
- 상담사 연결 여부 판단
- 고위험 질문에 대한 일반 답변 차단
- 금지 표현 검증
- 로그 저장 및 개선 데이터 축적

핵심 구조는 다음과 같습니다.

```text
사용자 질문
↓
질문 전처리
↓
온토로지 기반 시술명·문의유형·증상·위험도 분류
↓
라우팅/하네스 판단
↓
벡터 DB FAQ 검색
↓
답변 생성
↓
답변 안전성 검증
↓
최종 응답 또는 상담사 연결
↓
로그 저장
```

# 1. 하네스의 역할

## 1-1. 하네스란?

이 프로젝트에서 하네스는 **FAQ, 벡터 DB, 온토로지, LLM, 상담사 연결 규칙을 안전하게 묶어주는 실행 제어 레이어**입니다.

쉽게 말하면:

> 온토로지는 판단 기준표이고, 하네스는 그 기준표대로 챗봇이 실제로 움직이게 하는 제어 장치입니다.

## 1-2. 하네스가 필요한 이유

성형외과 FAQ 챗봇은 단순 쇼핑몰 챗봇과 다릅니다.

사용자가 다음처럼 질문할 수 있습니다.

```text
필러 맞고 입술이 하얗게 변했어요. 괜찮나요?
눈수술 후 앞이 흐릿하게 보여요.
코수술 후 코피가 계속 나요.
보톡스 맞고 숨쉬기가 답답해요.
```

이런 질문에 챗봇이 단순 FAQ를 검색해 다음처럼 답하면 위험합니다.

```text
붓기나 멍은 일반적으로 생길 수 있습니다.
며칠 지켜보세요.
정상적인 회복 과정일 수 있습니다.
```

하네스는 이런 위험한 일반 답변을 막고, 즉시 상담사 또는 병원 연락 안내로 라우팅합니다.

# 2. 전체 시스템 구성 요소

| 구성 요소 | 역할 |
|---|---|
| FAQ 문서 | 병원 안내, 비용, 수술 전후 주의사항 등 원본 지식 |
| 벡터 DB | 사용자 질문과 의미가 비슷한 FAQ 검색 |
| 온토로지 | 시술명, 증상, 위험도, 문의유형, 대응규칙 구조화 |
| 하네스 | 전체 질문 처리 흐름 제어 |
| LLM | 검색 결과와 규칙을 바탕으로 자연어 답변 생성 |
| 상담사 연결 모듈 | 고위험·예약·환불·복잡 문의를 상담실로 전달 |
| 로그 저장소 | 질문, 판단, 답변, 상담사 연결 여부 기록 |

# 6. 위험도 분류 기준

## 6-1. 위험도 레벨

| 위험도 | 의미 | 기본 처리 |
|---|---|---|
| low | 일반 안내 가능 | FAQ 자동 답변 |
| medium | 단정 금지, 상담 권장 | FAQ 답변 + 병원 확인 안내 |
| high | 부작용·응급 가능성 | 일반 답변 차단 + 즉시 병원 연락 |
| critical | 생명·시야·호흡 등 긴급 가능성 | 응급 안내 + 병원 즉시 연락 |

# 15. 하네스 처리 의사코드

```python
def handle_user_question(question: str) -> dict:
    normalized_question = normalize(question)

    if detect_sensitive_personal_data(normalized_question):
        return response_with_template("TPL-PRIVACY-BLOCK")

    procedure = classify_procedure(normalized_question)
    question_type = classify_question_type(normalized_question)
    symptoms = extract_symptoms(normalized_question)
    risk_level = determine_risk_level(procedure, question_type, symptoms)

    route = apply_routing_rules(
        procedure=procedure,
        question_type=question_type,
        symptoms=symptoms,
        risk_level=risk_level
    )

    if route.action in ["URGENT_CLINIC_CONTACT", "EMERGENCY_GUIDANCE"]:
        answer = render_template(route.response_template_id)
        return build_response(answer, route, human_escalation=True)

    search_results = vector_search(normalized_question, top_k=5)

    if search_results.top_score < 0.72:
        answer = render_template("TPL-ASK-MORE")
        return build_response(answer, route, human_escalation=True)

    draft_answer = generate_answer_with_llm(
        question=normalized_question,
        retrieved_faqs=search_results,
        ontology_context={
            "procedure": procedure,
            "question_type": question_type,
            "risk_level": risk_level,
            "forbidden": route.forbidden_response
        }
    )

    validated_answer = validate_answer_safety(
        draft_answer=draft_answer,
        risk_level=risk_level,
        forbidden_phrases=FORBIDDEN_PHRASES
    )

    log_interaction(
        question=question,
        procedure=procedure,
        question_type=question_type,
        risk_level=risk_level,
        route=route,
        answer=validated_answer,
        search_results=search_results
    )

    return build_response(validated_answer, route)
```

# 19. LLM 프롬프트용 시스템 지시문 샘플

아래 지시문은 LLM 답변 생성 단계에 사용할 수 있습니다.

```text
너는 소규모 성형외과의 고객응대 FAQ 챗봇이다.

너의 역할:
- 진료시간, 예약, 위치, 비용 범위, 수술 전후 일반 주의사항을 안내한다.
- 진단, 처방, 수술 가능 여부, 부작용 여부를 단정하지 않는다.
- 고위험 증상이 있으면 일반 FAQ 답변을 하지 않고 병원 연락 또는 응급 안내를 한다.

반드시 지켜야 할 규칙:
1. "정상입니다", "괜찮습니다", "기다려보세요"라고 단정하지 않는다.
2. 비용은 확정 금액으로 말하지 않는다.
3. 사진만 보고 수술 가능 여부를 판단하지 않는다.
4. 결과를 100% 보장하지 않는다.
5. 복용약 중단 또는 지속 여부를 직접 지시하지 않는다.
6. 고위험 증상은 즉시 병원 연락을 안내한다.

입력으로 제공되는 정보:
- 사용자 질문
```

# 23. 최종 결론

온토로지를 적용했다는 것은 챗봇이 다음을 이해할 수 있게 되었다는 뜻입니다.

```text
필러 + 피부색 변화 = 고위험
눈성형 + 시야 이상 = 고위험
비용 문의 = 확정 금액 단정 금지
사진 진단 요청 = 의료 판단 차단
환불 문의 = 상담실 연결
```

하지만 실제 하네스는 이 판단을 실행해야 합니다.

```text
고위험이면 일반 답변 차단
상담사 연결 버튼 표시
금지 표현 검증
FAQ 검색 신뢰도 확인
로그 저장
운영자가 개선할 수 있게 기록
```

따라서 실전형 구조는 다음과 같습니다.

```text
FAQ 문서
+ 벡터 DB
+ 온토로지
+ 라우팅 규칙
+ 답변 안전성 검증
+ 상담사 연결
+ 로그 저장
= 성형외과 실전형 AI 상담 하네스
```
